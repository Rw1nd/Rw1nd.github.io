March 08 2021

用实验室有几台新的服务器，为了能在服务器上配置的时候不翻车，自己先用虚拟机模拟一下搭建过程。

虚拟机环境如表：

Hostname ip 说明
k8s-master 192.168.193.132 server with GUI
k8s-node1 192.168.193.133 minimal install
k8s-node2 192.168.193.134 minimal install

master节点脚本（root用户运行）：

我用的是centos 7.9.2009，测试可以直接安装好dashboard和metrics-server。网速慢的时候需要等一会才可以运行dashboard，否则网页浏览报错。

安装完毕后可以直接从本地虚拟机的浏览器访问http://localhost:8001/api/v1/namespaces/kubernetes-dashboard/services/https:kubernetes-dashboard:/proxy/

如果要在物理机上访问，在物理机的terminal上用ssh转发ssh -L localhost:8001:localhost:8001 -NT abc@192.168.193.132
